# Future Workflow Ideas

These are workflow ideas to implement later as needed.

## Potential Workflows

- **`new-app-config.md`** — Full workflow for adding a brand new application config from scratch (create stow package, initial config, stow, test, commit)
- **`system-update.md`** — Post `pacman -Syu` checklist: check for deprecated config options, test critical apps (i3, picom, nvim), verify systemd services
- **`backup-restore.md`** — Steps for deploying dotfiles on a fresh Arch install (clone, install deps, stow all packages, verify)
- **`theme-change.md`** — Applying a new catppuccin flavor or switching themes across all apps (kitty, i3, nvim, rofi, dunst, polybar, gtk)
